记录：

What is “colour[u]” for?

Colour for the color map in “matplotlib” you choose, and the “u” means the tracer unit in “tru” (like 0, 1, 2) to distinguish different material.

Questions in Plotting

I drew a cumulative volume – depth graph roughly, show here below left. Compared with the fig in Peak-ring structure and kinematics from a multi-disciplinary study of the Schrödinger impact basin (shown below right), there are many differences. Some important point like the start and end points of range of depth are almost wrong.

Also, I think here are two ambiguous questions:

  • First is the technique to show the outcomes. The way I drew this graph is using a CDF (cumulative distribution function) function to realize it straightly. I don’t know whether this is a proper way to calculate CDF especially for discrete data.

  • Second one is about the method to measure. About the concept of “peak-ring material”. In the paper mentioned before, there are several lines : For the purposes of this plot, we defined the peak ring to be that material within 2 km of the surface and between radii of 80–100 km in the thin-crust model and 70–100 km in the thick-crust model, based on the position of the middle of the peak ring in each scenario. I use this radium range, but for the thickness I enlarge it to 0-20km. In the second aspect I have some questions:

Q1: How to define a surface’s coordinate?

Read the end time of simulation, then find the surface. The time can be read by “step.time” which is synchronize with real time in life.

Q2: Within 2km the number of tracers is about hundreds, I think it is not very enough for give the regular pattern for depth.

Trace how tracer moves in this impact process, see how it works. The number doesn’t means the volume, so maybe don’t mind.
Also, if the thickness of sampling does affect the accuracy of distribution, find out the relationship of accuracy and thickness (quantity), then choose the thickness in the range which in insensitive to the outcomes.

Q3: After setting the distance unit to km, dose the depth read-in is in km?

Yes, use “model.setScale” to set the unit.

Q4: The y-axis means the cumulative volume of peak-ring materials, how to calculate the volume of materials. Here I just use the number of tracers to measure. However, I don’t know whether it is correct.

The volume can’t be calculated by frequency, because one tracer means one annulus, the volume each tracer represents is not equal. “pySALEPlot” provide a method to get the volume of each tracer under tracer class – “tru”. Which can be found in the “init.py” and other files.

So, use the accumulated volume by dividing the depths into several bins to calculate. Then can get a true volume percentage in a certain range of depth:
percentage = volume under D km / volume of sample choose from peak ring

Changes

Use “model.MassVolume” to get the volume, then use “step.truvol()” to get volumes of each tracers.

Then normalized the volumes, get the CDF of depth.

The outcomes still not correct compared with Kring. I notice here are some problems I didn’t think of before and will optimize next week.

  1. How to change the size of each bins (with the for loop method I use)

  2. Are there any more effective algorithms for calculate CDF (functions in libs best)

  3. The exact stop times of simulation

  4. Ignore the volumes of impactor when calculate the origin of depth (maybe). Although most of the material in the peak-ring (within 2 km under the surface) is from impactor.

From this attempt I learned:

  1. Procedures to process the data (read the data first then do analyze).
  2. Simple usage of tracer
  3. Some simple functions to show the feature of data into figures (histogram, distribution)
    I realize that the post-processing is very likely to data analysis, especially I finished data mining course last semester.
  4. What the tracer means for (volume)
  5. Timestep
  6. Try to see the resource code first when don’t know
  7. How to handle the uncertainty and decrease the effect of one factor.
